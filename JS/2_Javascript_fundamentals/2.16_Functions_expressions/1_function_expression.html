There is another syntax for creating a function that is called a Function Expression. It allows us to create a new function in the middle of any expression.

In more advanced situations, that we’ll come across later, a function may be created and immediately called or scheduled for a later execution, not stored anywhere, thus remaining anonymous.

Function is a value

no matter how the function is created, a function is a value. Both prev examples store a function in the sayHi variable.

Please note that the last line does not run the function, because there are no parentheses after sayHi. There are programming languages where any mention of a function name causes its execution, but JavaScript is not like that.

Why is there a semicolon at the end?

The answer is simple: a Function Expression is created here as function(…) {…} inside the assignment statement: let sayHi = …;. The semicolon ; is recommended at the end of the statement, it’s not a part of the function syntax. The semicolon would be there for a simpler assignment, such as let sayHi = 5;, and it’s also there for a function assignment.
